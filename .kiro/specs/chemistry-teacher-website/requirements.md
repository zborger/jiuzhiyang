# 需求文档

## 概述

本文档定义了化学教师个人品牌与AI教学网站的需求规范。该网站将作为教师长期运营的内容管理、学生互动和未来AI教学功能的平台。系统采用演进式架构设计原则，支持功能的渐进式扩展，避免大规模重构。

## 术语表

- **化学教师网站**: 化学教师个人品牌和教学平台的主要Web应用系统
- **访客**: 任何未经身份认证访问网站的用户
- **内容项**: 包括文章、课程资料、PDF文档、PPT课件等任何教学资源
- **课程分类**: 教学内容的分类体系（如高一化学、高二化学、高三化学）
- **文章**: 教师发布的博客文章或教学文章
- **资源文件**: 可下载的教学资料（PDF、PPT、实验讲义等）
- **AI教学助手**: 未来AI驱动的题目解析和知识讲解功能
- **内容管理系统**: 未来用于管理网站内容的后台系统
- **知识库**: 未来基于教师课件和讲义构建的AI检索知识库

## 功能需求

### 需求 1: 网站基础与导航

**用户故事:** 作为访客，我希望能够轻松导航网站，以便快速找到所需信息。

#### 验收标准

1. 化学教师网站应在导航菜单中显示以下6个板块（按顺序）：首页、关于我、课程资源、教学博客、资料中心、联系方式
2. 当访客点击任意导航项时，化学教师网站应在300毫秒内导航至对应页面
3. 当访客在视口宽度小于768像素的设备上浏览时，化学教师网站应显示汉堡菜单图标，点击后展开显示所有6个导航选项
4. 当访客在视口宽度大于或等于768像素的设备上浏览时，化学教师网站应显示水平导航菜单，所有6个导航项可见且可点击
5. 化学教师网站应在所有页面显示相同的6个导航项，且顺序一致
6. 当汉堡菜单展开时，如果访客点击菜单外部区域，化学教师网站应自动收起菜单

### 需求 2: 首页展示

**用户故事:** 作为访客，我希望在首页看到教师简介概览，以便快速了解教师身份和提供的内容。

#### 验收标准

1. 化学教师网站应在首页展示教师的姓名、最小尺寸为200x200像素的照片、以及不超过500字符的介绍文字
2. 化学教师网站应在专属板块展示教师的教学理念，带有可见标题，文字长度不超过1000字符
3. 化学教师网站应展示主要课程方向：高一化学、高二化学、高三化学
4. 化学教师网站应在首页展示最近发布的3篇文章或资源作为精选内容
5. 如果文章或资源少于3篇，化学教师网站应展示所有可用内容，不显示占位符
6. 当访客点击精选内容时，化学教师网站应在300毫秒内导航至对应的详情页面
7. 如果精选内容的详情页面不可用，化学教师网站应显示错误提示，说明内容暂不可用
8. 化学教师网站应在4G网络连接测试下优化首页的Core Web Vitals指标（LCP < 2.5秒, FID < 100毫秒, CLS < 0.1）

### 需求 3: 关于我页面内容

**用户故事:** 作为访客，我希望了解教师的背景信息，以便评估教师的资质和信誉。

#### 验收标准

1. 当访客访问关于我页面时，化学教师网站应展示教师的教育背景，包括院校名称和学位，最多显示5条记录
2. 化学教师网站应以时间线格式展示教师的教学经历，最多显示20条记录，每条记录包含时间、学校、职位
3. 化学教师网站应展示所获奖项和资格证书，最多显示15条记录
4. 化学教师网站应展示教师的电子邮箱地址，格式符合标准邮箱格式规范
5. 化学教师网站应展示最小尺寸为150x150像素的微信二维码图片
6. 如果教师提供了社交媒体链接，化学教师网站应展示最多5个社交媒体链接（微博、抖音、B站等）

### 需求 4: 课程资源展示

**用户故事:** 作为访客，我希望按年级浏览课程资料，以便找到符合需求的学习资源。

#### 验收标准

1. 化学教师网站应展示课程分类：高一化学、高二化学、高三化学
2. 当访客选择某个课程分类时，化学教师网站应展示该分类下的所有内容项，每页最多显示50项
3. 化学教师网站应展示每个内容项的标题（最多100字符）、简要描述（最多200字符）和类型标识
4. 当访客选择筛选条件时，化学教师网站应按类型（文章、视频、文档）筛选内容项
5. 化学教师网站应默认按时间倒序展示内容项
6. 如果某个课程分类下没有内容项，化学教师网站应显示"暂无内容"提示
7. 如果访客选择了无效的筛选类型，化学教师网站应显示错误提示并保持当前列表不变

### 需求 5: 教学博客系统

**用户故事:** 作为访客，我希望阅读教学文章，以便学习化学概念和学习方法。

#### 验收标准

1. 化学教师网站应展示文章列表，包含标题、发布日期、分类和摘要（最多150字符）
2. 当访客点击文章标题时，化学教师网站应导航至文章详情页面
3. 化学教师网站应展示完整的文章内容，支持标题（H1-H3）、段落、代码块、图片、公式的正确渲染
4. 化学教师网站应支持以下文章分类：教学心得、考试分析、学习方法、实验知识
5. 当访客选择某个分类时，化学教师网站应仅显示该分类下的文章
6. 当访客在搜索框输入关键词（至少2个字符）时，化学教师网站应显示标题或内容包含该关键词的文章列表
7. 如果搜索结果为空，化学教师网站应显示"未找到相关文章"提示
8. 化学教师网站应在每篇文章详情页末尾展示最多3篇相关文章
9. 当访客在移动设备上浏览时，化学教师网站应以最小16像素的字体大小渲染文章正文

### 需求 6: 资源文件管理

**用户故事:** 作为访客，我希望访问可下载的学习资料，以便离线学习。

#### 验收标准

1. 化学教师网站应展示资源文件列表，包含标题、描述、文件类型、文件大小（MB或KB单位）
2. 化学教师网站应支持以下资源类型：PDF文档、PPT课件、实验讲义
3. 当访客点击资源下载按钮时，化学教师网站应在2秒内启动文件下载
4. 如果资源文件有预览图片，化学教师网站应显示最小尺寸为120x120像素的缩略图
5. 化学教师网站应按分类和类型分组展示资源
6. 如果资源文件不可用或损坏，化学教师网站应显示错误提示说明文件无法下载，并提供联系方式
7. 如果下载因网络错误失败，化学教师网站应显示错误提示并提供重试选项

### 需求 7: 联系信息展示

**用户故事:** 作为访客，我希望能够轻松联系教师，以便咨询问题或预约课程。

#### 验收标准

1. 化学教师网站应在主导航菜单中提供专门的联系页面入口
2. 化学教师网站应展示最小尺寸为200x200像素的微信二维码图片
3. 如果微信二维码图片加载失败，化学教师网站应显示替代文本，说明教师的微信号
4. 化学教师网站应将教师的电子邮箱显示为可点击的mailto链接
5. 化学教师网站应展示至少3个社交媒体主页链接（从微博、抖音、B站中选择）
6. 当访客在视口宽度大于1024像素的桌面设备上浏览时，化学教师网站应确保微信二维码、邮箱和社交媒体链接在初始视口内可见，无需滚动

### 需求 8: 响应式设计

**用户故事:** 作为使用各种设备的访客，我希望网站能够适应屏幕尺寸，以便获得最佳浏览体验。

#### 验收标准

1. 化学教师网站应使用Tailwind CSS断点（sm: 640px, md: 768px, lg: 1024px, xl: 1280px）实现响应式设计
2. 当访客在视口宽度大于1024像素的屏幕上浏览时，化学教师网站应显示完整桌面布局（三栏或两栏）
3. 当访客在视口宽度768px至1024px之间的屏幕上浏览时，化学教师网站应显示平板布局（两栏或单栏）
4. 当访客在视口宽度小于768px的屏幕上浏览时，化学教师网站应显示移动端布局（单栏）
5. 化学教师网站应确保触摸设备上所有交互元素（按钮、链接）的最小可点击区域为44x44像素
6. 化学教师网站应确保在视口宽度320px至1920px范围内，文本可读且无需横向滚动

### 需求 9: SEO优化

**用户故事:** 作为网站所有者，我希望网站对搜索引擎友好，以便潜在学生能够通过搜索引擎发现网站。

#### 验收标准

1. 化学教师网站应为每个页面设置唯一的meta标题，长度在30至60字符之间
2. 化学教师网站应为每个页面设置唯一的meta描述，长度在120至160字符之间
3. 化学教师网站应为每个页面实现Open Graph标签，包括og:title、og:description、og:image、og:url
4. 当内容被添加或更新时，化学教师网站应生成包含所有公开页面URL的sitemap.xml文件
5. 化学教师网站应实现标题层级结构，每个页面有且仅有一个H1元素，后跟按顺序排列的H2和H3元素
6. 化学教师网站应在关于我页面包含Person模式的JSON-LD结构化数据，在每篇博客文章页面包含Article模式的JSON-LD结构化数据
7. 化学教师网站应在所有页面使用语义化HTML元素，包括header、nav、main、article、section、footer
8. 化学教师网站应为每个页面实现指向首选URL的规范URL（canonical URL）链接元素

### 需求 10: 性能与加载速度

**用户故事:** 作为访客，我希望网站快速加载，以便无延迟地访问内容。

#### 验收标准

1. 化学教师网站在首页和所有公开页面应达到Lighthouse性能评分至少90分
2. 化学教师网站应对首屏以下的图片实现懒加载，当图片进入视口时才开始加载
3. 化学教师网站应使用Next.js Image组件优化所有图片，优先使用WebP格式；如果浏览器不支持WebP，则回退到JPEG或PNG格式
4. 化学教师网站应为每个页面路由实现代码分割，确保每个页面的JavaScript包大小不超过200KB（压缩后）
5. 化学教师网站应为所有公开页面启用静态页面生成（Static Site Generation）
6. 化学教师网站应为实现内容哈希文件名的静态资源设置缓存头，缓存期限为1年（31536000秒）
7. 当访客在3G网络条件下访问时，化学教师网站应在2000毫秒内返回初始HTML响应

### 需求 11: 内容数据结构

**用户故事:** 作为维护网站的开发者，我希望有结构良好的内容数据模型，以便未来添加功能时无需大规模重构。

#### 验收标准

1. 化学教师网站应定义Content_Item接口，包含以下字段：id（字符串类型）、title（最多100字符）、description（最多500字符）、type（枚举值）、category（字符串数组）、tags（字符串数组）、createdAt（ISO 8601日期时间字符串）、updatedAt（ISO 8601日期时间字符串）、content（字符串类型）
2. 化学教师网站应定义Article接口扩展Content_Item，额外包含字段：author（字符串类型）、publishDate（ISO 8601日期字符串）、readingTime（数字类型，单位分钟）、featured_image（URL字符串）
3. 化学教师网站应定义Resource_File接口扩展Content_Item，额外包含字段：fileUrl（URL字符串）、fileSize（数字类型，单位字节）、fileType（MIME类型字符串）、downloadCount（数字类型）
4. 化学教师网站应定义Course_Category接口，包含字段：id（字符串类型）、name（最多50字符）、description（最多200字符）、slug（URL友好字符串）、order（数字类型）
5. 化学教师网站应将所有内容数据存储在/content目录下，使用Markdown文件加YAML frontmatter格式，支持本地开发和未来CMS集成
6. 化学教师网站应实现getContent()、listContent()、searchContent()等内容加载工具函数，抽象数据源，支持未来迁移至数据库或CMS

### 需求 12: 无障碍访问合规

**用户故事:** 作为有无障碍访问需求的访客，我希望网站能够通过辅助技术导航，以便访问所有内容。

#### 验收标准

1. 化学教师网站应通过WCAG 2.1 Level AA合规检查，使用自动化工具（如axe-core）和手动测试验证
2. 化学教师网站应为所有有意义的图片（非装饰性图片）提供非空替代文本（alt属性）
3. 化学教师网站应确保所有交互元素（链接、按钮、表单控件）在获得焦点时有可见的焦点指示器（如轮廓线）
4. 化学教师网站应确保正常文本的颜色对比度至少为4.5:1，大文本（18px或14px粗体）的对比度至少为3:1
5. 化学教师网站应支持使用Tab键、Enter键、Escape键等键盘操作导航所有交互元素
6. 化学教师网站应为导航地标（nav、main、aside等）使用适当的ARIA角色标签（role="navigation"、role="main"等）
7. 化学教师网站应确保所有表单输入框有关联的label元素或aria-label属性

### 需求 13: 未来AI集成技术架构

**用户故事:** 作为开发者，我希望架构能够支持未来的AI功能，以便无需大规模重写即可添加。

#### 验收标准

1. 化学教师网站应在/app/api目录下实现API路由结构，用于未来后端功能
2. 化学教师网站应定义模块化组件结构，AI功能可作为独立组件添加到/components/ai目录
3. 化学教师网站应在/app/api/ai/analyze、/app/api/ai/explain、/app/api/ai/assistant路径创建占位API端点，返回501 Not Implemented状态
4. 化学教师网站应实现/lib/content-loader.ts数据层抽象，提供fetchContent()接口，支持未来RAG集成
5. 化学教师网站应使用环境变量（NEXT_PUBLIC_AI_API_URL、AI_API_KEY）配置外部AI服务端点
6. 化学教师网站应在/docs/ai-integration.md文件中记录AI功能集成的扩展点和接口规范

### 需求 14: 内容版本控制准备

**用户故事:** 作为网站所有者，我希望内容系统能够支持版本历史，以便未来能够追踪变更和恢复历史版本。

#### 验收标准

1. 化学教师网站应在所有内容数据结构的YAML frontmatter中包含createdAt和updatedAt字段，格式为ISO 8601（YYYY-MM-DDTHH:mm:ss.sssZ）
2. 化学教师网站应以Markdown + YAML frontmatter格式存储所有内容，便于Git版本控制集成
3. 化学教师网站应在每个内容文件的frontmatter中维护schemaVersion字段（如"1.0.0"），用于未来迁移支持
4. 当内容模式发生变更时，化学教师网站应在/scripts/migrate-content.ts提供迁移脚本

### 需求 15: 部署与基础设施

**用户故事:** 作为网站所有者，我希望网站易于部署，以便以最小的技术开销进行维护。

#### 验收标准

1. 化学教师网站应支持通过vercel deploy命令零配置部署到Vercel平台
2. 化学教师网站应支持使用Docker容器部署到腾讯云轻量服务器，通过docker run命令启动
3. 化学教师网站应在项目根目录提供Dockerfile，基于node:18-alpine镜像，暴露端口3000
4. 化学教师网站应使用.env.local、.env.staging、.env.production文件实现基于环境的配置
5. 化学教师网站应在README.md文件的"部署"章节提供详细的部署文档，包含Vercel和Docker两种方式
6. 化学教师网站应在/app/api/health路由实现健康检查端点，返回200状态和JSON响应{"status":"ok"}

### 需求 16: 错误处理与用户反馈

**用户故事:** 作为访客，我希望在出现问题时获得清晰的反馈，以便理解发生了什么并采取适当的行动。

#### 验收标准

1. 如果页面不存在（404错误），化学教师网站应显示自定义404错误页面，包含"页面未找到"提示和返回首页的链接
2. 如果发生服务器错误（500错误），化学教师网站应显示自定义500错误页面，包含"服务器错误"提示和重试按钮
3. 当资源（图片、文件）加载失败时，化学教师网站应显示有意义的错误提示，如"图片加载失败"或"文件下载失败"
4. 化学教师网站应在控制台记录所有错误，包含错误类型、错误消息、发生时间、页面URL等上下文信息
5. 当访客禁用JavaScript时，化学教师网站应显示基础内容（使用<noscript>标签），并提供"请启用JavaScript以获得最佳体验"提示

### 需求 17: 化学内容渲染

**用户故事:** 作为化学学生，我希望化学公式和方程式能够正确渲染，以便理解科学内容。

#### 验收标准

1. 化学教师网站应支持使用HTML下标（<sub>）和上标（<sup>）标签渲染化学式，如H₂O、CO₂
2. 化学教师网站应支持使用Unicode字符或KaTeX渲染带有反应箭头（→、⇌）的化学方程式
3. 化学教师网站应支持使用KaTeX库渲染数学方程，包括上标、下标、分数、根号、积分等
4. 化学教师网站应确保化学符号在Chrome、Firefox、Safari、Edge的最新两个主要版本上正确渲染
5. 化学教师网站应在/docs/content-formatting-guide.md文件中为内容作者提供化学符号格式化指南，包含示例和最佳实践
